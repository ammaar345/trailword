## Visual Design

### Marshmallow Palette (Warm Pink Family)
No longer grayscale. Tiles use warm marshmallow-inspired colors:
- `--tile-correct-bg`: `#f7d3d3` (light mode), `#c4a2a2` (dark mode) — soft marshmallow pink
- `--tile-present-bg`: `#edd5d5` (light mode), `#a8888a` (dark mode) — whisper pink
- `--tile-absent-bg`: `#ece8e4` (light mode), `#2a2826` (dark mode) — warm cream / near-black
- Text uses dark `#3a3030` on light tiles, white on dark tiles
- Colors defined as CSS custom properties in `src/index.css`

### Contrast Variants
5 levels selectable via Settings dialog, each changes BOTH tile colors AND full-page surface colors:
- **Medium** (default) — soft pink tile family, neutral gray page background
- **High** — higher contrast page background (pure white light / near-black dark), punchier key colors
- **Soft** — warm cream page background (`#fdfcfa` light / `#1a1814` dark), muted brown undertones throughout
- **Dark** — moody dark-leaning surfaces even in light mode, deep ink tones
- **Colorblind** — blue/amber/gray tiles for colorblind accessibility, neutral page surface (same as Medium)

Each variant overrides `--color-surface-*` and `--key-*` CSS variables + marshmallow-header/card gradients.

### Marshmallow UI Elements
All interactive elements use pillowy marshmallow styling (14px border-radius, soft gradients, layered box-shadows):
- **Keyboard keys**: `key-marshmallow` + `key-marshmallow-default` classes in `src/index.css`
- **Action buttons** (Hint/Share/Stats/Practice/Settings): Blue marshmallow variant `marshmallow-btn-blue` — light blue gradient `#d6e8ff→#c0daff` (light mode), `#2a3d66→#1f3050` (dark mode)
- **Header bar**: `marshmallow-header` class — per-contrast gradient, sticky, backdrop blur
- **Clue card**: `marshmallow-card` class — per-contrast gradient

### Winning Tile Text
Correct tile text color driven by `--tile-correct-text` per contrast variant. No hardcoded color.

### Dither Texture
Bayer ordered-dither pattern overlays non-empty tiles via `opacity-30`. Generated as SVG data URI in `src/lib/dither-pattern.ts`.

### Fonts
Font stack in `src/index.css`:
- `--font-sans: "Inter"` — body
- `--font-display: "Space Grotesk"` — headings/category
- `--font-serif: "Instrument Serif"` — accent/italic
- No external font files needed (Google Fonts via CSS import)

### Grid
- 1px tile borders
- Gaps: 1 (mobile), 1.5 (desktop)
- Max width: 300px

## Word Validation

### Accepted Guesses
Any 5-letter alphabetic word that isn't a swear word. Validation logic in `src/lib/swear.ts`:
- Checks: length === 5, only a-z letters, not in swear word set, in VALID_GUESSES set
- `VALID_GUESSES` built as union of dictionary (15,921 5-letter words from dwyl/english-words) + answer words (1,281)
- Generated by filtering `words_alpha.txt` (370K total) → 5-letter → remove swear → merge with answer words
- File: `src/lib/valid-guesses.ts` (auto-generated, 143 KB)
- Swear word filter blocks ~14 common offensive words

### Answer Words
1,281 curated words with categories and hints. Defined in `src/lib/words.ts`:
- Category distribution: Ideas(278), Traits(165), Objects(115), Science(81), People(77), Nature(73), Place(68), Life(67), Food(65), Arts(58), Body(57), Mind(52), Animals(39), Time(23), Travel(22), Tech(15), Stories(14), Colors(10), Sports(2).

## Sound System

### Web Audio API (ASMR keyboard)
File: `src/lib/sounds.ts`
- **key** — short noise burst + low thump (tactile profile)
- **backspace** — slightly different timbre (linear profile)
- **enter** — fuller thunk (clicky profile)
- **correct** — ascending two-tone chime
- **wrong** — low descending tone
- **win** — three-note ascending arpeggio C5-E5-G5-C6
- **lose** — soft descending two-tone
- **click** — very short tick (UI elements)

Settings: enabled toggle + volume slider (persisted in `localStorage['trailword:prefs']`)

## Design Decisions
- Colors are CSS variables in `src/index.css` — Tailwind surface colors (`--color-surface-*`), tile colors, and key colors all overridable per contrast variant
- Tile/keyboard status CSS variables use `var(--name, fallback)` pattern for resilience
- No framework for state management — React `useState`/`useCallback` only
- No routing library — simple page components with conditionals
- Daily puzzle seeded from date for deterministic daily rotation
